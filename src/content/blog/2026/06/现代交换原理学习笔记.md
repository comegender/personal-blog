## chapter1 交换概论

### 什么是交换

在通信网上，负责在通信的源和目的终端之间建立通信信道传送通信信息的机制

### 通信网三要素

交换设备、传输设备、终端设备

### 通信网的分层结构

**应用层**

**业务层**

+ 电路交换

  + PSTN(电话网)、B-ISDN、PSPDN、2G（GSM，CDMA）、DDN（数字数据网）

+ 分组交换

  + 4G，5G，Internet

+ 多速率电路交换

+ 帧中继

  + 帧中继网

+ 电路&分组

  + 2.5G，3G

+ 智能网：

  呼叫控制于业务处理物理分离产生智能网

  业务生成于业务处理分开——快速提供业务

  业务传输/交换与业务处理分开——集中提供业务

**支撑网**

+ 信令网
  + 通信网的**“神经系统”**
+ 数字同步网
  + 保证网络中的交换和传输等设备的时钟同步

  + 完成电信网和电信业务的性能管理、配置管理、故障管理

### 交换方式

+ CTM（电路交换）

  + 三个阶段：建立连接->信息传送->释放连接
  + 特点：
    + 面向连接的工作方式
    + 同步时分复用
    + 无差错控制
    + 对通信信息不做处理
    + 流量控制基于呼叫损失制
  + 多速率电路交换：将几个基本信道捆绑起来，速率是基本速率的整数倍
    + 保证时隙顺序的完整性：因为电路交换对通信信息是不做处理的，不保证时隙顺序会导致收到的信息顺序错乱或者模糊

+ ATM（异步交换模式）

  + MPLS、软交换、IMS
  + 信元长度固定，可用硬件实现，有利于简化交换控制和缓冲器管理
  + 短信元可以减少交换节点内部的缓冲器容量以及排队时延和时延抖动
  + 特点：
    + 面向连接
    + 异步时分复用
    + 极大提高了网络传输的处理能力

  | 6. 可靠性 & 实时性        | 实时性：极高，时延恒定，无抖动；可靠性：链路故障直接中断通话，无迂回路由，可靠性低。 | 实时性：最差，时延抖动剧烈；可靠性：每个分组独立路由，链路中断时分组可绕行，可靠性高。 | 实时性：中等，时延稳定，抖动小；可靠性：虚电路断连需要重新建路，分组不能自动绕行，可靠性弱于数据报。 | 实时性：很高，转发时延低、抖动极小，可承载实时音视频；可靠性：支持 LSP 快速重路由，链路故障毫秒级切换，实时性与可靠性兼顾。 |

- 排列表达式：$\begin{pmatrix}t1\quad t2\quad  \cdots\quad  tn\\r1\quad  r2\quad  ⋯\quad rn\end{pmatrix}$，入线排列表达式$(0,1,2...,N−2,N−1)$

### S接线器

S 和 T 接线器仅用于 PCM（数字）交换。

- 构成：
  - 交叉点矩阵（开关阵列）。
  - 一组控制存储器（CM，Control Memory）。
- 功能：输入输出线之间的空间交换。
- 规格：
  - 一个 S 接线器所含控制存储器 CM 的数量=入（出）线数。（输入控制 CM 数=入线数，反之）
  - 每个控制存储器 CM 含有的存储单元个数=时隙数。
  - 每个存储单元中存储索引存储单元的地址，为m位 bit，且满足N≤2m，其中N为接线器的入（出）线数。
- CM 具有输入控制方式与输出控制方式：
  - **输入控制：控制一条输入线与每一条输出线的交叉点的开启与闭合，存储单元的内容表示对应输入复用线与哪一个输出复用线的交叉点开关在该存储单元所对应的时隙内接通**（对于入线a，在当前时隙$TS_i$，从$CM_a$的i号存储单元读出需要交换到的出线b，打开输入线a与b相交叉的开关，完成空间交换）
  - **输出控制：控制一条输出线与每一条输入线的交叉点的开启与闭合，存储单元的内容表示对应输出复用线上哪个交叉点在该存储单元所对应的时隙内接通**（对于出线b，在当前时隙$TS_i$，从$CM_b$的i号存储单元读出接受输入的入线a，打开输入线a与b相交叉的开关，完成空间交换）

![img](/img/modern-switching/2022-06-12-12-34-36-S6SqHv.png)

> 其中0-n-1代表的是时隙数（多少路PCM）

+ 特性
  + 只完成空间交换，不进行时隙交换
  + 空间接线器按时分方式工作
  + 输出控制方式易于实现多播
  + 输出控制方式可自动避免出线冲突

### T接线器

时分交换单元一般分为：共享存储器型交换单元、共享总线型交换单元。

- 共享存储器型：共N个出入线缓存。入线缓存控制读出、出线缓存控制写入。
- 共享总线型：总线按照时隙轮流分配给入线控制部件和出线控制部件使用。
  - $kNV=B/T$（N为入线数、V为入线速率、B为每时隙传送比特数、T为时隙长度，k为时隙分配因子）

**异同**

+ 相同点： 

  （1）两种时分交换单元内部都只有一条唯一的通路，由各子信道分时共享；

+ 不同点： 

  （1）共享存储器型内部唯一的公共通路是存储器，共享总线型内部唯一的公共通路是总线； （2） 共享存储器型的存储器被划分成 N 个区域，N 路输入信号被放在存储器的 N 个区域中， 不同区域的 N 路信号被读出形成 N 路输出信号；共享总线型则按照总线控制权的分配方法，把总线的工作时间划分为多个时段（最简单的是划分为 N 个时隙），在每个时段内把总线分给某条入线；当输入部件获得总线上的输入时隙后把入线的信号送到总线上，同时，对应的输出部件将总线上的信息读入并从出线上输出信息； 

  （3）共享存储器型交换速度与控制器有关，共享总线型交换速度与总线宽度有关。

**时间接线器的构成：**

- 话音存储器（SM，Speech Memory）：暂存话音的数字编码信息。
  - 存储单元个数等于 PCM 线路每帧时隙数；每单元至少 8bit（一路话音）。
- 控制存储器（CM，Control Memory）：
  - 存储单元个数等于 SM；每单元至少$log_2n$（n为时隙数）比特。

**时间接线器输出控制方式：顺序写入、控制读出。**

- SM：前半周期顺序写入话音、后半周期从地址读出话音

- CM：前半周期顺序写入控制地址、后半周期控制读出话音。

  缺点：复用级别越高，对控制电路要求越高，无法达成很大规模；时隙转换延迟不定。

![img](/img/modern-switching/2022-06-12-12-42-16-hFPJ1s.png)

**时间接线器输入控制方式：控制写入、顺序读出。**

![img](/img/modern-switching/2022-06-12-12-42-27-gIdw4p.png)

特性：

- 时分交换，用于交换同步时分复用的信号。
- SM 划分为N个区域，每个区域一个字节，存放一个话音数据。各个区域间不共享，无排队缓冲。
- 交换的控制过程由 CM 硬件实现，速度快。
- 交换时延小且稳定。
- 严格无阻塞，并且可避免出线冲突。
- 易于实现同发和广播。
- 速率固定。

### 交换网络基本要素

**交换单元**、不同交换单元间的**拓扑连接**和**控制方式**

**分类**

+ 单级交换网络

  + 由一个或者多个位于同一级的交换单元所构成的交换网络，并且当同一级有多个交换单元构成时，不同交换单元入线和出线之间可建立连接

    ![1778216449190](/img/modern-switching/1778216449190.png)

+ 多级交换网络

  ![1778216542808](/img/modern-switching/1778216542808.png)

**交换网络的阻塞**

是指从交换网络不同输入端来的信息在交换网络中交换时发生了对同一公共资源争抢的情况，这是在竞争中失败的信息就会被阻塞，竞争有内部竞争和出线竞争

+ 严格无阻塞网络：只要需要交换的入线与出线空闲，就可以通过交换网络建立一个连接
+ 可重排无阻塞网络：只要需要交换的入线和出线空闲，就可以通过交换网络直接的或对已有连接进行重排之后建立一个网络
+ 广义无阻塞：只要需要交换的入线和出线空闲，不必重排，就可以通过交换网络建立连接

### CLOS网络

3 级 CLOS 严格无阻：m≥2n−1

3 级 CLOS 可重排无阻：m≥n

构造法：根据要求确定中间级个数m，一三级分别采用n×m和m×n交换单元，r=N÷n（一三级单元个数），中间级采用r×r交换单元。

### BANYAN网络

**特性**：单通路，有阻塞(内部阻塞)，基于树形结构所以能自选路由

构造法：构造N×N的 BANYAN 网络，需要$log_2N$级，每级N/2个2×2交换单元，共$(N/2)×log_2N$个。拿两个n×n网络，将n个2×2交换单元作为一级通过混洗连接到前部、或者通过反转混洗连接到尾部。

![img](/img/modern-switching/2022-06-12-12-57-41-cMWPcm.png)

降低阻塞的方法

+ 增加级数

+ 引入排序网络

+ 限制入线上的信息量，加大缓冲存储器

+ 构成多通道交换网络（增加平面、增加连接）

### BENES网络

两个 BANYAN 网络背靠背连接，并将中间两级合并为一级。是**可重排无阻塞网络**，**多通路**。 构造法：N×N的 BENES 网络需要$(N/2)×(2×log_2{N−1})$个2×2交叉单元。

#### 基本概念

+ 边缘路由器LER
  + 入口LER：分配标记、将IP分组封装成标记分组、将标记分组从相应端口转发出去
  + 出口LER：去除标记，将标记分组还原为IP分组；查找路由表将IP分组转发出去
+ 核心路由器：LSR
  + 依据标记信息表进行转发
+ 标记交换路径LSP
  + MPLS网络为具有一些共同特性的分组通过网络选定的一条通路，由入口的边缘交换路由器，一系列核心路由器和出口的边缘交换器以及它们之间由标记所标识的逻辑信道组成
+ 标记分发协议LDP
  + MPLS的控制协议，用于LSR之间交换信息，完成LSP的建立、维护和拆除等
+ 转发等价类FEC
  + 在MPLS网络中经过相同的LSP，完成相同的转发处理的一些数据分组，这些数据分组具有某些相同的特性；FEC的划分通常依据网络层的目的地址前缀或是主机地址
+ 标记映射
  + 将标记分配给FEC

## chapter5 移动交换

### 移动通信解决制约的关键技术

“蜂窝”概念的提出解决了移动通信中频率资源有限的问题。

+ 频率复用

  + 蜂窝移动系统再给定区域内部署基站，利用**信号功率随传播距离衰减**的特点，在**不同空间区域实现频率重复使用**，提高频谱效率

    + 每个蜂窝基站被分配一组频段，这些频段在一个小的地理区域内使用，该区域被成为蜂窝，也称为小区
    + 将若干个相邻小区组成一个区群；每个区群都可使用整个蜂窝系统的全部频段，这些频段被划分为子频段供一个区群内的不同小区使用，相邻小区的基站分配不同的子频段
    + 不同区群中对应的小区使用相同子频段，即在一个给定的覆盖区域内，存在许多使用同一组频段的小区，同频小区之间存在同频干扰

  + 频率复用直接决定了小区间干扰，并因此决定了系统的容量和性能

    ![1782139971799](/img/modern-switching/1782139971799.png)

  + 无线信道（频段）是移动通信中移动台与基站之间的信息信道，可进一步分为逻辑信道、传输信道与物理信道

    + 信道最重要特征参数是信息传递能力，也称信息通过能力
    + 在典型的高斯信道条件下，信道的信息通过能力与**通过频带宽度、工作时间以及信号功率与噪声功率之比**密切相关。频带越宽、工作时间越长、信号与噪声功率比越大，信道的通过力就越强。

+ 移动性管理

  + 移动性管理是移动通信系统区别与固定通信的核心技术之一，其本质是通过**位置追踪、无缝切换、高效寻呼**，实现终端在移动中通信不中断、服务不掉线
  + 移动性管理的功能贯穿于移动通信的接入网和核心网，不同代际通信系统存在差异，单核心功能一致
    + 位置管理：是移动性管理的基础，解决网络如何知道终端在哪的问题，确保移动台能被有效的寻呼到。主要包括两个主要任务：**位置登记、呼叫传递**。对应的两个问题：**位置更新和寻呼**
      + 当用户进入一个信道位置登记区域，将进行位置更新，位置更新解决的问题是“移动台如何发现位置变化及合适报告它的当前位置”
      + 寻呼是移动通信网络在不知道用户确切位置时，发起的一种全网广播或区域广播机制，目的是解决网络如何找到空闲状态终端的问题
    + 切换管理： 借助切换功能实现网络实体跨覆盖区的无缝业务衔接
      + 切换是指在无线通信系统中，当移动台从一个小区移动到另一个小区时，为了保持移动用户在移动过程中跨越不同无线覆盖区域时通话不断，需要进行的信道转换
      + 三个问题：越区切换标准，即合适需要进行越区切换；越区切换如何控制；越区切换时的信道分配

  2. 鉴权请求。AMF转发UE身份信息给AUSF和UDM请求对用户进行合法性检查，即鉴权
  3. 进行合法检查（双向鉴权）。UDM确认用户购买过5G服务，指示AUSF/AMF和UE进行双向鉴权
  4. 获取签约数据和网络使用要求，鉴权成功后，AMF分别从UDM和PCF处获取用户签约数据以及用户用网策略。后续网络基于此为用户分配用网资源
  5. 接收用户注册请求。AMF通知UE初始注册完成，可以接入网络

+ PDU会话建立流程

  + 5G中的PDU会话是用户设备UE与数据网络DN之间建立的端到端逻辑连接，用于传输分组数据单元PDU，是5G网络中数据通信的核心载体

  + 作用

    + 建立数据通路：为UE与目标DN之间提供双向数据传输通道
    + 承载QoS需求：根据不同业务对时延、带宽、可靠性的要求，分配相应的服务质量等级
    + 绑定网络切片：每个PDU会话都归属与特定的网络切片，实现资源隔离与专属保障

  + 基本流程

    1. UE经由无线基站发送PDU会话建立请求信息给AMF，请求建立数据通路
    2. AMF查阅UE基本信息，并将UE的请求上报到SMF
    3. SMF到PCF处获取会话建立策略，包括会话质量要求、计费策略、变更通知策略
    4. SMF选择合适的UPF，为建立PDU会话提前做好信令准备
    5. SMF通知无线基站和UE，按指定要求建立PDU会话上行通道
    6. SMF通知UPF，按指定要求建立PDU会话下行通道

### SDN的产生

**传统路由器/交换机与服务器的对比**

|            |        设备配置与管理        |                           操作系统                           |             硬件系统             |                 芯片                 |
| ---------- | :--------------------------: | :----------------------------------------------------------: | :------------------------------: | :----------------------------------: |
| 传统路由器 | 静态手动配置，特性迭代速度慢 | API接口少，仅支持命令行界面（封闭式操作系统）；不具备外部可编程性 |          锁定特定供应商          | 创新周期缓慢；成本高昂，缺乏规模经济 |
| 服务器     |  网络启动；集中式配置与管理  |            开源或闭源；虚拟化或裸机；支持多种模型            | 激烈的竞争；品牌产品或“白盒产品” |            竞争与快速创新            |

**服务器**：开放的架构：软硬件分离，开发新业务灵活

**路由器**：封闭的设备：软硬件绑定，开发新功能困难

#### 数据平面

SDN交换设备没有二层交换机、路由器、三层交换机之分，基本功能包括转发决策、背板转发、输出链路调度

+ 转发决策：用流表代替了传统网络设备二层和三层转发表。数据分组进入SDN交换机后先与流表进行匹配查找，若与其中一个表象匹配成功，则执行相应处理动作；否则上交控制器，由其决定处理决策
+ 背板转发：通过背板把各端口连接起来，数据分组转发决策后，经背板从入端口转发到出端口
+ 输出链路调度：SDN交换机对QoS的支持主要有基于流表项设置报文入队列、根据meter进行限速，基于counter进行计费、基于group的select功能进行队列调度等

  #### 南向接口

  南向接口协议式控制器与交换机间信息交互的关键，其设计目标主要包括：

  + 收集数据平面信息，下发控制策略，指导转发行为
  + 实现网络配置与管理
  + 执行路径计算，涵盖链路属性、状态和拓扑信息

  ![1782220281830](/img/modern-switching/1782220281830.png)

  + openflow协议是用来描述控制器和openflow交换机之间数据交互所用信息的接口标准，其核心是openflow协议信息的集合
  + 三种消息类型:
    + 控制器-交换机：由控制器发起，用来管理或获取openflow交换机的状态
    + 异步：由openflow交换机发起，用来将网络事件或交换机状态变化更新到控制器
    + 对称：双向对称消息，用于建立连接和检测对方是否在线

+ NETCONF协议
  + netconf在SDN中主要用于对网络设备进行配置管理和状态监控。采用XML编码配置数据和RPC机制，与YANG数据模型紧密结合，成为实现网络自动化和集中管理的关键技术之一
  + 协议允许SDN控制器以编程的方式统一配置网络设备
  + 协议还能实时获取设备的运行状态和统计信息
  + NETCONF通常与YANG数据建模语言配合使用
  + 在SDN的多租户环境中，NETCONF可用于动态创建和配置虚拟网络资源

#### 北向接口

+ 北向接口是SDN架构中连接应用层与控制层的桥梁，它为开发者提供了一组基于网络高层逻辑抽象与业务模式的编程接口。通过这套接口，开发者能够实现对网络行为的灵活定义和动态调度
+ 从标准化角度看，开放网络基金会（ONF）定义的北向接口致力于提供一种与厂商无关的抽象网络视图。它将底层网络资源封装为统一的服务模型，使上层应用无需关心具体设备的差异，从而实现业务意图的快速部署与跨平台胡同
+ 在具体实现层面，REST API使目前最为广泛的北向接口形态
